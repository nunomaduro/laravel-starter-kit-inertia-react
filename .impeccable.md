# Impeccable Design Context

> **Canonical source of truth is DESIGN.md** — this file provides a quick-reference summary for impeccable integration.

## Design Context

### Users
Mixed audience — developers and non-technical end users (SaaS teams, org admins, content editors). UI should evoke **delight** — enjoyable, modern, rewarding.

### Brand Personality
**Clean, Sharp, Professional** — Linear/Stripe energy. Precise, modern, no-nonsense.

### Aesthetic Direction
- **Industrial-Minimal** — function-first, sharp edges, restrained decoration
- **Dark-first** — dark mode is the hero experience
- **References**: Linear, Raycast, Vercel
- **Anti-references**: Generic Bootstrap/Material, glassmorphism, oversized corners, decorative gradients

### Typography
- **Display/Headings**: JetBrains Mono 700 — monospace headings signal "built by developers"
- **Body**: IBM Plex Sans 400/500/600 — clean, technical, data-dense optimized
- **Code**: JetBrains Mono 400 — unified with display font
- **Scale**: 40/32/24/20 (JetBrains Mono) → 16/14/12 (IBM Plex Sans) → 11 (JetBrains Mono labels)
- **Loading**: Google Fonts `IBM+Plex+Sans:wght@400;500;600&JetBrains+Mono:wght@400;500;600;700`

### Color System
- **Accent**: Muted teal `oklch(0.65 0.14 165)` — sophisticated, not screaming
- **Neutrals**: Cool blue undertone (hue 260) in all grays
- **Background (dark)**: `oklch(0.11 0.005 260)`, **Surface**: `oklch(0.15 0.005 260)`
- **Semantic**: Success (145°), Warning (85°), Destructive (25°), Info (250°)
- **No card shadows** — background-level differentiation only

### Accessibility
- **WCAG 2.1 AA** — 4.5:1 body text, 3:1 large text/UI components
- Full keyboard navigation, screen reader support, `prefers-reduced-motion` respected

### Design Principles
1. **Precision over decoration** — every element earns its place
2. **Speed is a feature** — skeleton states, optimistic updates, smooth transitions
3. **Dark-first confidence** — design dark first, adapt for light
4. **Keyboard-first, mouse-friendly** — command palette, shortcuts, focus rings
5. **Progressive disclosure** — show what matters now, reveal complexity on demand

### Motion
- Minimal-functional: 100ms micro, 200ms transitions, 300ms page-level
- `ease-out` entrances, `ease-in` exits. No bounce, no spring.

### Component Style Notes
- `//` comment-prefix section labels (monospace, uppercase, accent color)
- Badges: monospace lowercase, pill shape
- Buttons: subtle, accent with dark text for primary
- Tables: monospace data cells, hover row highlight
- Alerts: colored dot + text, no icons
