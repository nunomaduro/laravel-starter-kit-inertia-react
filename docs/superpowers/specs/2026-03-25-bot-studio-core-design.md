# Bot Studio Core — Design Spec (Spec A)

> **Date**: 2026-03-25
> **Timeline**: Flexible — take the time to get it right
> **Approach**: Vertical slices (data layer → builder UI → knowledge pipeline → templates → plan-gating)
> **Scope**: Agent Builder + Runtime + Knowledge/RAG + Templates (core product)
> **Out of Scope**: Marketplace (Spec B), Embed Widget (Spec C)
> **Module**: `modules/bot-studio/`

---

## Overview

Bot Studio lets users create custom AI agents without code — guided wizard for persona, tool picker, knowledge file upload (RAG), and a split-screen builder with live preview. "GPTs for any SaaS" — built as a module, org-scoped, plan-gated.

### What It Replaces

v3 "Bot in a Box": 11 categories, 46 bots, 481 prompt templates, external service dependency (botinabox.online + botmysql). Admin-only creation, no RAG, no custom tools.

v4 Bot Studio: self-service creation, built-in execution (laravel/ai), RAG knowledge base (pgvector), tool selection (ModuleToolRegistry), plan-gated, fully org-scoped.

### Dependencies (all built)

- `OrgScopedAgent` — agent wrapper with tenant context
- `ModuleToolRegistry` — collects and filters AI tools from modules
- `SemanticSearchService` — pgvector similarity search
- `model_embeddings` table with `chunk_index` support
- `HasVisibility` trait for visibility/sharing
- Spatie Media Library for file storage
- `spatie/pdf-to-text` (poppler-utils) for PDF extraction
- `phpoffice/phpword` for DOCX extraction

### agent_conversations modification

Add `agent_definition_id` (nullable FK to `agent_definitions`, nullOnDelete) to the existing `agent_conversations` table. NULL = global assistant chat (backward compatible).

### How knowledge embeddings work

Knowledge file chunks go into the existing `model_embeddings` table:
- `embeddable_type` = `AgentKnowledgeFile`
- `embeddable_id` = knowledge file ID
- `chunk_index` = 0, 1, 2, ... (one per chunk)
- `organization_id` = agent's org

Knowledge files do NOT use the `HasEmbeddings` trait. The `ProcessKnowledgeFileJob` handles extraction → chunking → embedding as its own pipeline. This keeps the trait simple (single-embedding per model) and knowledge processing as a separate concern.

When the agent searches, `SemanticSearchService` is scoped to:
1. `organization_id` = current org
2. `embeddable_type` = `AgentKnowledgeFile`
3. `embeddable_id` IN (knowledge file IDs for this agent)

Each agent's knowledge is isolated — Agent A cannot search Agent B's files.

## 3. Builder UI

### Create Page — Guided Wizard (`/bot-studio/create`)

5-step wizard with progress bar:

**Step 1: Identity**
- Name (auto-generates slug)
- Description
- Avatar upload (Spatie Media)
- Visibility selector (private / organization — via HasVisibility)

**Step 2: Persona**
- Role input: "What role should this agent play?"
- Tone picker: tag-style selector (Professional, Friendly, Casual, Technical, Empathetic)
- Expertise: "What topics should it know about?"
- Restrictions: "What should it refuse to do?"
- Live-generated system prompt preview below the form
- "Edit manually" link opens raw prompt editor (escape hatch)

The `PromptWizardService` generates the system prompt from wizard answers. Answers stored in `wizard_answers` JSON column so users can re-enter the wizard later.

**Step 3: Tools**
- Multi-select from `ModuleToolRegistry::getToolsForOrganization()` (plan-filtered)
- Each tool shows name + description
- Tools not available on current plan shown greyed out with upgrade prompt

**Step 4: Conversation Starters**
- Add/remove suggested first messages
- Shown as clickable chips in the chat UI

**Step 5: Review**
- Summary of all choices
- Model picker dropdown (from `BotStudioSettings::allowed_models`)
- Temperature slider (0.0 - 1.0)
- "Create Agent" button → saves to DB, redirects to edit page

### Edit Page — Split-Screen Builder + Live Preview (`/bot-studio/{slug}/edit`)

**Left panel (50%):** Tabbed editor
- **Prompt tab**: System prompt textarea with variable insertion buttons (`{{org_name}}`, `{{user_name}}`, `{{current_date}}`), model picker, temperature slider. "Re-run Wizard" button re-opens the guided flow with saved answers.
- **Tools tab**: Same multi-select as wizard step 3
- **Knowledge tab**: File upload (drag-and-drop), file list with status badges (Queued/Processing/Indexed/Failed), retry button on failures, delete with confirmation. Storage usage indicator.
- **Settings tab**: Visibility, max_tokens, other config
- **Starters tab**: Conversation starter management

**Right panel (50%):** Live preview chat
- Green "Live Preview" indicator
- Agent avatar + name
- Conversation starters shown as clickable chips
- Full chat experience — changes to prompt/tools reflect immediately without saving
- Ephemeral messages (not saved to conversation history, not billed)
- "Clear chat" button to reset

**Top bar:** Agent name + status badge (Draft), "Re-run Wizard" button, "Save" button.

### My Agents List (`/bot-studio`)

- Grid layout of agent cards
- Each card: avatar, name, tool/knowledge count, description, visibility badge, chat count
- "X of Y agents used" counter (from plan limits)
- "Create Agent" + "Browse Templates" buttons
- Empty state with template suggestions

## 4. AgentRunner Service

```
modules/bot-studio/src/Services/AgentRunner.php
```

Configures an `OrgScopedAgent` from an `AgentDefinition` at chat time. No new agent class — composition over inheritance.

**Responsibilities:**

1. **Load definition** — Fetch `AgentDefinition` by slug, verify access via `HasVisibility` scope
2. **Resolve system prompt** — Replace variables (`{{org_name}}`, `{{user_name}}`, `{{current_date}}`) via `Str::replace()`
3. **Resolve tools** — Take `enabled_tools` JSON array, filter through `ModuleToolRegistry::getToolsForOrganization()` to ensure org still has access
4. **Add RAG tool** — If agent has indexed knowledge files, inject a scoped `KnowledgeSearchTool` filtered to this agent's `AgentKnowledgeFile` IDs
5. **Configure OrgScopedAgent** — Set custom system prompt, filtered tools, page context
6. **Return streaming response** — Delegates to OrgScopedAgent's `stream()` method

**Fluent API:**

```php
$runner = app(AgentRunner::class);
$stream = $runner
    ->forDefinition($agentDefinition)
    ->withUser($user)
    ->withContext($pageContext)
    ->stream($prompt);
```

**What it does NOT do:**
- No conversation management — stays in the controller
- No credit deduction — middleware on OrgScopedAgent
- No template logic — templates are just definitions with `is_template = true`

### KnowledgeSearchTool

```
modules/bot-studio/src/Ai/Tools/KnowledgeSearchTool.php
```

A laravel/ai tool that wraps `SemanticSearchService` scoped to a specific agent's knowledge files:
- Constructed with the agent's knowledge file IDs
- Searches only those embeddings
- Returns results with source file name citations
- Implements `ModuleAiTool` with `requiredFeature()` returning `'bot_studio'`

## 5. Knowledge Pipeline

### ProcessKnowledgeFileJob

```
modules/bot-studio/src/Jobs/ProcessKnowledgeFileJob.php
```

Async job dispatched on file upload:

1. **Update status** → `processing`
2. **Extract text** based on mime type:
   - PDF: `spatie/pdf-to-text` (wraps `pdftotext` CLI via poppler-utils)
   - DOCX: `phpoffice/phpword`
   - TXT, CSV, MD: read directly
3. **Chunk** via `DocumentChunker` — ~500 tokens per chunk, 50-token overlap
4. **Embed** — Send each chunk to embedding API (provider from `config/ai.php`)
5. **Store** — Upsert into `model_embeddings` with `embeddable_type=AgentKnowledgeFile`, `embeddable_id=file_id`, `chunk_index=0,1,2...`
6. **Update status** → `indexed` with `chunk_count` and `processed_at`
7. **On failure** → `failed` with `error_message`

Rate-limited via `spatie/laravel-rate-limited-job-middleware`.

### DocumentChunker

```
modules/bot-studio/src/Services/DocumentChunker.php
```

- Splits text into chunks of ~500 tokens
- 50-token overlap between chunks for context continuity
- Returns array of `{text, chunk_index}` pairs
- Token counting via simple word-based approximation (1 token ≈ 0.75 words)

### KnowledgeProcessor

```
modules/bot-studio/src/Services/KnowledgeProcessor.php
```

Orchestrates the upload flow:
- Validates file type and size against `BotStudioSettings`
- Stores file via Spatie Media Library on the `AgentKnowledgeFile` model
- Creates the `AgentKnowledgeFile` record with `status = pending`
- Dispatches `ProcessKnowledgeFileJob`

Handles deletion:
- `AgentKnowledgeFile` model registers a `deleting` event that bulk-deletes from `model_embeddings` where `embeddable_type = AgentKnowledgeFile` and `embeddable_id = file_id`
- Removes the Spatie Media file

## 6. Templates System

### Sources

**Starter kit seeds generic templates** via `BotStudioTemplateSeeder`:
- General Assistant — all-purpose, no tools pre-selected
- Customer Support — empathetic tone, FAQ-oriented prompt
- Data Analyst — analytical tone, suggests chart/table tools
- Onboarding Guide — friendly, step-by-step walkthrough prompt

Each template has pre-filled `wizard_answers`, `system_prompt`, `conversation_starters`, and suggested `temperature`/`model`.

**Modules register domain-specific templates** via `ProvidesAgentTemplates`:

```php
// modules/bot-studio/src/Contracts/ProvidesAgentTemplates.php
interface ProvidesAgentTemplates
{
    /** @return array<int, array{name: string, description: string, system_prompt: string, wizard_answers: array, conversation_starters: array, enabled_tools: array, model: string, temperature: float}> */
    public function agentTemplates(): array;
}
```

Modules implement this on their `ModuleProvider`. `BotStudioModuleServiceProvider` collects templates from all providers at boot (same pattern as `ModuleToolRegistry`) and ensures they exist in the DB as `is_template = true`, `organization_id = null` records.

### "Use this template" flow

1. User browses `/bot-studio/templates` — grid of template cards
2. Clicks "Use this template"
3. `AgentDefinitionController@duplicate` copies the definition into user's org (`is_template = false`, `organization_id = user's org`)
4. Redirects to edit page — user owns the copy and can customize everything
5. `wizard_answers` copied too — user can re-enter the wizard to tweak

### Out of scope (MVP)

- No template categories/tags
- No template versioning
- No "update from template" sync

### BotStudioSettings


|----------|------|---------|-------|
| `max_agents_basic` | int | 3 | Limit for basic plan tier |

| `max_knowledge_file_size_mb` | int | 10 | Per file |
| `max_knowledge_total_mb` | int | 100 | Per agent |
| `default_model` | string | 'gpt-4o-mini' | Default for new agents |
| `allowed_models` | array | ['gpt-4o-mini', 'gpt-4o', ...] | Models users can pick |

Added to `SettingsOverlayServiceProvider::OVERLAY_MAP` (`orgOverridable: true`). Filament system page at `ManageBotStudio.php`.

### Feature Flag

```php
// config/feature-flags.php — feature_metadata
'bot_studio' => [
    'label' => 'Bot Studio',
    'description' => 'Create custom AI agents',
    'delegate_to_orgs' => true,
    'plan_required' => 'pro',
],
```

### Enforcement Points

1. **Feature gate** — `/bot-studio/*` routes behind `BotStudioFeature` Pennant middleware
2. **Agent count** — `AgentDefinitionController::store()` checks current count vs. plan tier limit from `BotStudioSettings`. Returns 403 with upgrade prompt at limit.
3. **Knowledge limits** — `KnowledgeFileController::store()` validates file size and total per agent against settings
4. **Model access** — Builder's model picker only shows models from `allowed_models` setting

All routes under `tenant` + `auth` + `feature:bot_studio` middleware:

### Agent Chat

| Method | Route | Action | Notes |
|--------|-------|--------|-------|
| POST | `/bot-studio/{slug}/chat` | `AgentChatController@stream` | Stream via AgentRunner (persistent, billed) |
| POST | `/bot-studio/{slug}/preview` | `AgentChatController@preview` | Stream via AgentRunner (ephemeral, not saved, not billed — skips credit middleware) |
| GET | `/bot-studio/{slug}/conversations` | `AgentChatController@conversations` | List conversations |

### Knowledge Files

| Method | Route | Action | Notes |
|--------|-------|--------|-------|
| POST | `/bot-studio/{slug}/knowledge` | `KnowledgeFileController@store` | Upload (enforces limits) |
| DELETE | `/bot-studio/{slug}/knowledge/{id}` | `KnowledgeFileController@destroy` | Delete + embeddings |
| POST | `/bot-studio/{slug}/knowledge/{id}/retry` | `KnowledgeFileController@retry` | Re-process failed |

### Key Design Decisions

- **Slug-based URLs** — cleaner, shareable
- **Separate chat endpoint** (`/bot-studio/{slug}/chat`) from global `/api/chat` — uses `AgentRunner` to configure from definition
- **`duplicate`** handles both template forking and agent cloning
- **`agent_definition_id`** on `agent_conversations` links conversations to specific bots (NULL = global assistant)

- [ ] Custom system prompt with variable resolution
